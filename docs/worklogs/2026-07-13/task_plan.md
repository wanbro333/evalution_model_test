# 信贷决策模型重构实施计划

## 目标

按照已确认的设计规格，完成从原始附件到监督违约概率、TOPSIS解释、期望收益优化、疫情调整、真实灵敏度分析以及统一报告输出的可复现流水线。

## 已确认决策

- 正则化逻辑回归PD为主模型，TOPSIS仅作辅助解释。
- 原始信誉D级和预测D级均禁贷。
- 期望利润包含留存、违约损失、资金成本。
- 问题2和问题3预算10000万元，单户额度0或10至100万元。
- 原始附件保持不变。
- 论文扩写目标18至22页，采用分层伴读式结构。
- 网页定位为“论文伴读 + 结果展示”，不改变模型定义和正式结果。

## 遇到的错误

| 错误 | 尝试次数 | 解决方案 |
|---|---:|---|
| `rg.exe` 在环境中被拒绝访问 | 2 | 使用PowerShell文件枚举与Select-String |
| bundled Python缺少matplotlib/scipy/sklearn | 1 | 使用当前系统Python并补充项目依赖清单 |
| 默认Python缺少scikit-learn | 1 | 使用NumPy/SciPy自行实现正则化逻辑回归与交叉验证，避免在线依赖 |
| 设计文档首次暂存被只读.git权限拒绝 | 1 | 获得审批后成功暂存和提交 |
| 截图生成命令末尾使用错误相对路径核对文件 | 1 | 截图实际已生成，改用绝对路径检查 |
| 合并补丁因进度日志文本未精确匹配而中止 | 1 | 拆分源代码与日志补丁后成功应用 |
| 从`paper`子目录运行网页渲染器导致模块导入失败 | 1 | 切换到`信贷决策建模`目录后重新生成 |
| 浏览器截图写入与紧随其后的文件核对存在短暂竞态 | 2 | 截图命令后单独使用绝对路径读取，不把即时核对作为失败条件 |
| 首轮移动端宽度修复后标题仍被grid子项最小宽度撑开 | 1 | 为hero及其直接子项设置`min-width:0`，标题启用强制中文换行 |
| PowerShell将`$base?target=...`误解析为变量名 | 1 | 改用完整字面URL传递验收查询参数 |
| 无头Chrome在iframe内锚点滚动时出现首屏未滚动或局部空白 | 2 | 验收页改为克隆目标章节和正式CSS到顶部，不再依赖iframe滚动 |
