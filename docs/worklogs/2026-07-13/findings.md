# 重构发现与约束

## 原始问题

- 负数有效发票被取绝对值，附件1进项和销项累计额分别放大约11.73%与9.80%。
- 附件2独立归一化并独立生成TOPSIS理想解，无法继承附件1风险尺度。
- 原始信誉D级24家中11家获得贷款，合计约817.48万元。
- 灵敏度权重未参与TOPSIS重算，论文灵敏度图由围绕0.965的随机数生成。
- 流失率仅事后展示，未进入额度、利率或期望收益决策。
- 论文、图表和Dashboard存在多组硬编码旧统计。
- 行业识别中187/302家为“其他”，56家名称仅为个体经营编号。

## 数据结构

- 附件1：123家企业；进项210947条、销项162484条；27家违约。
- 附件2：302家企业；进项395175条、销项330835条。
- 附件3：29个有效利率点，4%至15%，A/B/C三档流失率。
- 发票日期覆盖约2016-10至2020-02。

## 实施约束

- 样本较小，主模型优先稳定、可解释和概率校准，不追求复杂黑箱。
- 所有训练变换必须在训练折内拟合；附件2只使用附件1最终参数。
- 报告数字必须由机器生成的统一指标文件提供。

## 环境确认

- 默认Python为3.14.0，具有NumPy 2.5.1、Pandas 3.0.3、SciPy 1.18.0、Matplotlib 3.11.0、openpyxl 3.1.5。
- SciPy提供`optimize.milp`，可实现额度混合整数规划。
- 默认Python没有scikit-learn，监督模型与验证指标使用NumPy/SciPy原生实现。
- TeX Live 2026提供XeLaTeX和latexmk；系统未发现LibreOffice命令。

## 新模型基线

- 连续月历和净额年化特征在附件1、附件2分别生成43列和41列，无NaN或无穷值。
- 重复分层嵌套交叉验证ROC-AUC为0.8557，PR-AUC为0.7339，Brier分数为0.1066。
- A/B/C/D档样本外实际违约率依次约5.4%、12.5%、26.7%、80.0%，具有严格单调性。
- 最终L2系数为0.1；附件1档位分布56/32/15/20，附件2档位分布157/68/37/40。

## 新策略基线

- 单调流失率拟合修复A/B/C曲线中的3/2/2个下降点，拟合RMSE均低于0.002。
- 问题1预算8000万元、80家放贷、原始D级贷款为0、组合期望净收益约70.54万元。
- 问题2预算10000万元、100家放贷、组合期望净收益约133.56万元。
- 疫情冲击作用于赔率后8家等级变化，问题3预算仍为10000万元，期望净收益约132.19万元。
- 100次真实TOPSIS权重扰动Spearman均值0.9983、最小0.9958；移除年化利润影响最大，Spearman约0.9150。

## Dashboard重设计审计

- 现有页面依赖Google Fonts与ECharts CDN，离线环境无法保证字体和图表加载。
- 页面使用多套强调色、大量卡片和衬线标题，信息层级偏展示页，不适合评审快速核查模型证据。
- 违约率、收益、灵敏度和图表序列直接写死在HTML，已经与当前结果漂移。
- 重构采用单文件静态HTML、系统中文字体、单一青绿色强调色、生成PNG图表和真实表格；保留原主要导航主题，不引入前端包。

## 最终产物验证

- 最终样本外验证：ROC-AUC 0.8557、PR-AUC 0.7339、Brier 0.1066、Log Loss 0.3571。
- 最终策略：问题1额度8000万元、80家、期望净收益70.54万元；问题2额度1亿元、100家、期望净收益133.56万元；疫情调整后100家、期望净收益132.19万元，8家跨等级。
- 固定PD档位实际违约率为A 5.36%、B 12.50%、C 26.67%、D 80.00%；原始D级获贷企业数为0。
- 100次TOPSIS真实扰动Spearman均值0.9983、最小0.9958；移除影响最大指标“年化利润”后Spearman为0.9150。
- 7项自动测试全部通过，Excel公式与引用检查通过，11页论文PDF无版式警告。

## 论文与网页二次改版审计

- 当前论文约9581字符、278行、11页，方法公式正确，但多数小节只有一至三段，缺少问题分析、建模流程图、符号总表、假设解释、逐步推导、算例和“结果意味着什么”的口语化说明。
- 监督PD、嵌套交叉验证、先验校正、TOPSIS固定尺度、保序回归和MILP之间的前后依赖没有用统一流程图串联，初学者容易把它们看成相互独立的方法堆叠。
- 当前网页只有模型验证、信贷策略、疫情调整、稳健性四段，适合快速查结果，但不足以解释数据如何变成特征、PD如何进入利率和额度决策。
- 网页目前采用单文件离线HTML和真实PNG图表，数据来源一致性良好，应保留离线、无CDN、统一指标源和现有锚点等优点。
- 二次改版应按“先直觉、再公式、再算法、再结果解读、最后局限”的教学节奏组织，而不是简单增加篇幅。
- 新网页采用原生HTML/CSS独立渲染器，避免继续扩大 `reporting.py` 的职责；原结果生成函数保留为legacy但主流程不再引用。
- 新增真实图表均可由现有输出计算，不需要修改风险模型：发票数据规模、两附件PD分布、三问收益拆分。

## 二次改版视觉验收

- 论文首页、PD验证页和MILP策略页的文字、公式、表格与教学提示框均清晰，无裁切。
- 首轮抽查发现第5页流程图第二行节点与连线重叠，已改为上下两排主流程，并以青绿色虚线单独标示TOPSIS解释支线。
- 首轮手机截图发现导航滚动区的flex子项未允许收缩，造成页面整体横向撑宽并裁切标题；已为导航添加最小宽度约束和移动端折行保护。
- 第20页原本只剩一条参考文献，视觉上过空；参考文献已改为紧凑小字号排版，目标是使正文自然收束到19页左右。

- 最终桌面与390px手机首屏均无横向溢出；手机导航采用品牌与横向导航双层布局，标题、流程卡和指标卡正常换行。
- 风险验证章节的两张图、固定档位表与交叉验证侧栏，以及额度策略章节的收益拆分宽表和解释提示均通过独立截图验收。
- 最终论文为19页，抽查首页、流程页、PD验证页、MILP页和结论参考文献页，未发现重叠、裁切或空白尾页。
